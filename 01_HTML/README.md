## üß© Core Principles Covered

### 1. HTML Document Anatomy

Understanding mandatory structural tags:

* `<html>`
* `<head>`
* `<body>`

This is non-optional knowledge.

### 2. Semantic HTML5

Using elements **based on meaning**, not appearance:

* `<article>`
* `<section>`
* `<aside>`
* `<nav>`

Semantics exist for browsers, screen readers, search engines, and future maintainability.

### 3. Metadata & SEO Basics

Proper use of:

* `<title>`
* `<meta charset>`
* `<meta name="description">`

SEO starts at the HTML layer.

### 4. Content Structuring

Correct usage of:

* Lists
* Tables
* Images
* Hyperlinks

Structure before styling. Always.

### 5. Form Fundamentals

Capturing user input with:

* `<form>`
* `<label>`
* `<input>`

Accessibility and clarity matter more than visuals.

### 6. Accessibility (A11y) First

Baseline rules:

* Always use `alt` on images
* Always bind labels using `for` and `id`

Accessibility is not optional. It‚Äôs engineering hygiene.

## üî∞ Tips for Beginners (Intellectual Sparring)

* **HTML is not a programming language**
  It does not implement logic. It structures information. Confusing this is a rookie mistake.

* **Validate your HTML**
  Use the W3C Markup Validation Service regularly. Browsers are forgiving; standards are not.

* **Semantics over aesthetics**
  Ignore how things look. Focus on what they *are*. CSS comes later.

* **Document flow matters**
  HTML is rendered top to bottom. This linear flow underpins everything in frontend engineering.

Truth beats convenience. Every time.
